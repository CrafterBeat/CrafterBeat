## What should you build with 2 hours today?

**🔍 What to Build Today**

**Why I'm Not Picking Other Candidates**

**Today's Build: A Cross-Network Relay Layer for LocalSend + Resumable Transfers**

Concrete product shape: a lightweight relay service that makes LocalSend work between devices on different local networks, while adding resumable transfers. The specific stack: write the WebSocket relay server in **Go** (drawing on the lightweight Go middleware thinking behind CJackHwang/ds2api on GitHub Trending — not reusing its code, just the approach). The core is maintaining a `session_id → chunk_offset` Redis Hash: the client writes the offset after each chunk upload, and on interruption resumes from that offset. Step one is spinning up a `/relay/{session_id}` endpoint with `gorilla/websocket` — both devices connect in, the server pipe-forwards between them, chunk size fixed at 256 KB, one ACK per chunk before writing to disk. Step two is implementing File API chunked reads on the frontend (PWA — following [@chr15m](https://news.ycombinator.com/item?id=47915917)'s PWA suggestion, which fits this scenario perfectly), querying Redis for the offset after reconnection and resuming from there. The entire MVP needs no account system — session_id is a 6-digit random code, shared by QR scan or just saying it out loud. Deploy on **Fly.io**, the free tier is sufficient. Pricing: $0 for self-hosted open source, $5/month for the hosted version (unlimited bandwidth, server stays yours).

**Weekend Extension Path**

After the 2-hour MVP validates that cross-network transfer works, the weekend focus is two things. First, align the relay layer with LocalSend's protocol — LocalSend uses HTTPS + mDNS, so the relay version needs to use the same JSON handshake format, otherwise LocalSend clients can't connect directly. Second, add end-to-end encryption — use **libsodium**'s `crypto_box` for key exchange, so the relay server only ever forwards ciphertext, cutting off privacy concerns entirely. That's also a direct answer to the sentiment stirred up by the Flo data scandal. The full product's differentiation story writes itself: LocalSend is LAN-only, Blip (recommended by [@nightwalkerkg](https://news.ycombinator.com/item?id=47935545)) requires an account, this product is **no-account cross-network** — three distinct, non-overlapping positions.

**Key Judgment**: The product form is a cross-network relay layer for LocalSend. Stack is Go + gorilla/websocket + Redis + Fly.io. Pricing is $5/month for the hosted tier. LocalSend on the HN front page today provides a distribution window, and the two explicit pain points in the comments — resumable transfers + cross-network support — are ready-made product copy.

**Counter View**: LocalSend is an actively maintained open-source project. The resume-on-interruption issue may already have a PR open in its GitHub repo — this is the prior-art risk. Open-source communities sometimes ship fixes faster than an indie product can launch. Check LocalSend's open issues before you start building.
